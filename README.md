# Week 2: Entity and Relationship Analysis

This directory contains the deliverables for the **E-Commerce Order Management Database System** project, specifically focusing on the conceptual and logical data modeling phase.

## Overview
As part of the ongoing development of the E-Commerce Order Management System, this week's activities involved identifying data requirements, defining entity attributes, establishing relational integrity, and mapping cardinality for the database schema.

## Deliverables

| File Name | Description |
| :--- | :--- |
| `Entity_Analysis_Report.pdf` | Detailed report identifying 9 core entities, their attributes, domain constraints, and primary/foreign key specifications. |
| `Entity_Relationship_Analysis.pdf` | In-depth analysis of inter-entity relationships, cardinality mapping (1:1, 1:N, M:N resolution), and referential integrity strategies (cascading actions). |

## Project Details
* **Project Title:** E-Commerce Order Management Database System
* **Context:** Data Requirements Analysis & Logical Database Design
* **Key Achievements:**
    * Identification of 9 normalized entities (3NF).
    * Mapping of complex business rules into relational cardinality.
    * Definition of database-level constraints (NOT NULL, UNIQUE, CHECK, DEFAULT).
    * Strategy for M:N relationship resolution using associative entities (`ORDER_ITEM`, `REVIEW`).
    * Specification of referential integrity and cascading rules to protect data consistency.
